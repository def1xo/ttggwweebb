# Color model training pipeline (2/3-color products)

Ниже — **рабочая схема**, чтобы не гадать “куда кидать архивы и что запускать”.

## 1) Что собирать

Для каждой картинки нужна строка с метаданными:

- `image_url` или локальный путь,
- `supplier`,
- `product_id` / `sku` (если есть),
- целевой цвет в вашей таксономии (`black`, `black-white`, `black-white-red`, ...).

Рекомендуемый формат: `CSV` (`dataset_items.csv`) или `JSONL`.

---

## 2) Массово скачать изображения

В репо уже есть параллельный загрузчик:
- `backend/scripts/downloader.py`

Пример для большого объёма (в т.ч. 9k+ на поставщика):

```bash
python backend/scripts/downloader.py \
  --input dataset_items.csv \
  --url-column image_url \
  --supplier-column supplier \
  --output-dir ./dataset/raw \
  --workers 48 \
  --timeout 25 \
  --max-retries 5 \
  --report download_report.csv
```

Что получаете:
- папки `dataset/raw/<supplier>/...`,
- `download_report.csv` (ok/fail, причина, путь).

---

## 3) Очистка датасета (обязательно)

Минимум:
- убрать битые/очень маленькие файлы,
- удалить дубли,
- убрать нерелевантные фото (баннеры, упаковка без товара, мемы и т.п.).

Практика:
- сначала выбросить `ok=0` из `download_report.csv`,
- затем ручная выборочная проверка по каждому поставщику.

---

## 4) Разметка для 2/3-цвета

Нужно размечать **мульти-лейбл** (а не один цвет):
- `color_1`, `color_2`, `color_3` (пустые допустимы),
- либо единый `palette_key` типа `black-white-red`.

Важно:
- Размечаем **цвет товара**, а не фон.
- Для кроссовок желательно отдельно отмечать основной upper/body (если делаете сегментацию).

---

## 5) Сплит и баланс

Разделение:
- train: 80%
- val: 10%
- test: 10%

Ограничения:
- один и тот же `sku`/модель не должен попадать в train и test одновременно,
- выравнивайте редкие комбинации (например `black-white-red`), иначе модель будет "схлопывать" всё в `black-white`.

---

## 6) Базовый цикл обучения

1. Берёте ваш датасет,
2. обучаете multi-label модель,
3. на `val/test` считаете F1 по каждому цвету и по комбинациям,
4. экспортируете предикт в ключ вашей системы (`black-white-red`),
5. включаете в импортер с fallback на текущий rule-based детектор.

Минимальный production-порог:
- macro-F1 >= 0.82 на test,
- для ключевых комбинаций (`black-white`, `black-white-red`) F1 >= 0.88.

---

## 7) Как запускать в Docker (без "куда это всё кидать")

Вариант:
- храните датасет на хосте в `./dataset`,
- монтируете в training-контейнер как volume,
- артефакты модели сохраняете в `./artifacts`.

Пример идеи запуска:

```bash
docker run --rm -it \
  -v $(pwd)/dataset:/workspace/dataset \
  -v $(pwd)/artifacts:/workspace/artifacts \
  your-training-image:latest \
  bash
```

Дальше внутри контейнера запускаете ваш train-скрипт на `/workspace/dataset`.

---

## 8) Что делать прямо сейчас

1. Подготовьте `dataset_items.csv` (url + supplier + label).
2. Прогоните `downloader.py`.
3. Пришлите:
   - `download_report.csv`,
   - 200–500 строк примера разметки,
   - вашу текущую таксономию цветов (какие combo разрешены).
4. После этого можно собрать конкретный `train.py` под ваш стек и подключить в импорт.
